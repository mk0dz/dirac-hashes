What is Dirac Hashes?

Dirac Hashes is a quantum-resistant cryptographic library designed to secure your applications against both classical and quantum attacks. Our algorithms are designed with the future in mind, providing robust security that can withstand the computational power of quantum computers.

Key Features:

  • Quantum-Resistant Hash Functions - Multiple hash algorithms designed to resist quantum-computing attacks
  • Post-Quantum Signatures - Including Dilithium (NIST standard), SPHINCS+, and Lamport schemes
  • Key Encapsulation Mechanisms - For secure key exchange in a post-quantum world
  • Simple API - Easy to integrate into your existing applications and blockchain systems

Basic Usage

The core functionality is divided into three main modules:

1. Hash Functions
from quantum_hash.dirac import DiracHash # Generate a hash with the default algorithm message = b"Your message here" digest = DiracHash.hash(message) # Use a specific algorithm digest = DiracHash.hash(message, algorithm="improved_grover") # Get a list of available algorithms algorithms = DiracHash.get_supported_algorithms()
2. Digital Signatures
from quantum_hash.signatures.dilithium import DilithiumSignature # Create a signer signer = DilithiumSignature(security_level=1, hash_algorithm="improved") # Generate keypair private_key, public_key = signer.generate_keypair() # Sign a message message = b"Message to sign" signature = signer.sign(message, private_key) # Verify a signature is_valid = signer.verify(message, signature, public_key)
3. Key Encapsulation Mechanism (KEM)
from quantum_hash.kem.kyber import KyberKEM # Create a KEM instance kem = KyberKEM(security_level=1) # Generate keypair private_key, public_key = kem.generate_keypair() # Encapsulate - generates a shared secret and ciphertext shared_secret, ciphertext = kem.encapsulate(public_key) # Decapsulate - recovers the shared secret from ciphertext recovered_secret = kem.decapsulate(ciphertext, private_key) # Both shared_secret and recovered_secret should be identical assert shared_secret == recovered_secret

API Reference

Our RESTful API is available at: https://dirac-hashes.onrender.com

Hash Endpoints

  • POST /api/hash/generate - Generate a hash digest
  • POST /api/hash/compare - Compare multiple hash algorithms
  • GET /api/hash/algorithms - List available algorithms

Signature Endpoints

  • POST /api/signatures/keypair - Generate a signature key pair
  • POST /api/signatures/sign - Sign a message
  • POST /api/signatures/verify - Verify a signature
  • GET /api/signatures/schemes - List available signature schemes

KEM Endpoints

  • POST /api/kem/keypair - Generate a KEM key pair
  • POST /api/kem/encapsulate - Encapsulate a shared secret
  • POST /api/kem/decapsulate - Decapsulate a shared secret
  • GET /api/kem/schemes - List available KEM schemes

For complete API documentation, visit our API Documentation page.

Security Level Guide

Our cryptographic primitives offer different security levels:

  • Level 1: Equivalent to AES-128
  • Level 3: Equivalent to AES-192
  • Level 5: Equivalent to AES-256

Choose based on your security requirements.

Performance Considerations

When implementing our library, consider:

  • Higher security levels require more computational resources
  • Signature schemes have different tradeoffs in terms of key size, signature size, and speed
  • The "improved" hash algorithm offers the best performance for most use cases

About Our Hash Algorithms

Dirac Hashes offers several quantum-resistant hash algorithms:

Improved

Our baseline algorithm with optimized performance and strong resistance to quantum attacks.

Grover

Specifically designed to resist Grover's algorithm quantum attacks.

Shor

Enhanced security against factorization attacks using Shor's algorithm.

Hybrid

Combines multiple approaches for balanced security and performance.

Improved Grover
